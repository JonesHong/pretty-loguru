pretty_loguru.core.configure_logger(level='INFO', log_path=None, component_name='', rotation='20 MB', subdirectory=None, log_name_format=None, timestamp_format=None, log_file_settings=None, logger_instance=None, service_tag=None, isolate_handlers=True, logger_format='<green>{time:YYYY-MM-DD HH:mm:ss}</green> | <level>{level: <8}{process}</level> | <cyan>{extra[folder]}</cyan>:<cyan>{function}</cyan>:<cyan>{line}</cyan> - <level>{message}</level>')[source]

Configure the logging system to ensure that each logger instance’s handlers are completely isolated.

Parameters:

  • level (Literal['TRACE', 'DEBUG', 'INFO', 'SUCCESS', 'WARNING', 'ERROR', 'CRITICAL']) – Log level, default is the global variable LOG_LEVEL.

  • log_path (Union[str, Path, None]) – Log storage path, defaults to the default path if None.

  • component_name (str) – Handler ID, used to tag log files.

  • rotation (Union[str, int]) – Log rotation size, in MB.

  • subdirectory (Optional[str]) – Subdirectory name, used to categorize different types of logs.

  • log_name_format (Optional[str]) – Log file name format.

  • timestamp_format (Optional[str]) – Timestamp format, used to customize time display.

  • log_file_settings (Optional[Dict[str, Any]]) – Other settings for log files.

  • logger_instance (Optional[Any]) – The log instance to initialize, uses the global _logger if None.

  • service_tag (Optional[str]) – Service name, used in log file names with the {service_tag} variable.

  • isolate_handlers (bool) – Whether to isolate handlers of different logger instances, default is True.

  • unique_id (#) – Unique ID, used to ensure each logger instance is completely independent.

  • logger_format (str) – Log format string, defaults to LOGGER_FORMAT.

Returns:

The full path of the log file.

Return type:

str
