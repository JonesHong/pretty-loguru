Contribution Guide

Thank you for considering contributing to the Pretty Loguru project! This page explains how to get involved in project development.

Development Environment Setup

# 克隆儲存庫
git clone https://github.com/yourusername/pretty-loguru.git
cd pretty-loguru

# 創建虛擬環境（推薦）
python -m venv venv
source venv/bin/activate  # Linux/macOS
venv\\Scripts\\activate  # Windows

# 安裝開發依賴
pip install -e ".[dev]"

Code Style

This project follows the PEP 8 code style guide and uses Black for code formatting:

# 在提交前格式化代碼
black pretty_loguru

Testing

Before submitting changes, ensure all tests pass:

# 運行所有測試
pytest

# 運行特定測試模組
pytest tests/test_core.py

# 檢查代碼覆蓋率
pytest --cov=pretty_loguru

Documentation Contribution

The documentation is generated using Sphinx:

# 安裝文檔依賴
pip install sphinx sphinx-rtd-theme sphinx-autodoc-typehints myst-parser

# 構建文檔
cd docs
make html

# 在瀏覽器中查看文檔
# Linux/macOS
open build/html/index.html
# Windows
start build\\html\\index.html

Contribution Process

  1. Fork the repository

  2. Create your feature branch: git checkout -b feature/amazing-feature

  3. Commit your changes: git commit -m ‘Added an amazing feature’

  4. Push to the branch: git push origin feature/amazing-feature

  5. Open a Pull Request

Bug Reports

If you find a bug, please report it in GitHub Issues and include the following information:

  • A brief description of the issue

  • Steps to reproduce

  • Expected behavior and actual behavior

  • Environment information (OS, Python version, etc.)

  • If possible, provide a minimal code example to reproduce the issue

Feature Requests

If you have an idea for a new feature, please propose it in GitHub Issues and provide as much of the following information as possible:

  • Feature description

  • Why this feature is valuable to the project

  • Suggestions for implementation (if any)

  • Possible usage examples

Changelog

When adding new features or fixing bugs, add the corresponding entry in CHANGELOG.md.

License

By submitting code, you agree to license your contribution under the project’s license.
